## 当前口径快照

### 抓取

- 正式 VE 主流程使用目标日 UI 日期范围，默认按 `target_date ~ target_date` 查询。
- 最新创意默认走 DOM 点卡 detail 主路径，raw 阶段尽量拿完整详情。
- 目标日素材仍以本地 `first_seen == target_date` 校验为准，避免 UI 夹带非目标日结果。
- 日常主流程不再额外跑二次 `dom_enrich`，该模块保留为排障工具。

### 分析

- 分析结果面向下游字段，而不是长正文报告。
- 关键结构化字段包括：Hook、脚本/口播、核心卖点、玩法指纹、模板指纹、差异点、玩法资产 ID/名称、玩法变种 ID/名称、玩法归类、玩法判断理由、风险标签、风险等级。
- `AI分析结果` 仍保留兼容短摘要，保证旧同步判定不因长正文下线而失效。

### 新素材判断

- 稳定玩法资产未见过：`新玩法`。
- 稳定玩法已见过，但模板指纹首次出现：`老玩法新迭代`。
- 稳定玩法和模板指纹都见过：`老玩法换皮`。
- 同模板只替换人种、肤色、性别或男女模特，不应进入新素材推送。

### 多维表

- 主表重点复核字段：玩法资产、玩法变种、玩法新旧、玩法资产 ID、玩法变种 ID、玩法判断来源、玩法判断理由、模板指纹、狭义新判断、狭义新理由、日内相似素材数。
- 「日内相似素材数」只比较同一天、同 appid 的素材；数值包含当前素材，也包含被同日封面 CLIP 聚类并到该代表组的素材，`1` 表示当天没有同类相似素材。
- 玩法重复默认不再让主表过瘦，更多成功分析素材会同步到主表，用字段辅助人工筛选。
- 每日抓取阶段会生成产品级保留漏斗，重点看 `clicked_detail_rows`、`captured_materials`、`kept_after_cover_filter` 和 `removed_reasons`。
- 全流程报告会继续串联分析入队、可用分析、同步候选和主表写入；异常低量时先提示人工决定是否按报告中的单产品命令重试。

## 下一阶段建议

1. 连续跑 3 到 5 天 VE 日报，人工标记狭义新判断是否符合预期。
2. 在多维表按「日内相似素材数」倒序复核，确认同一天扎堆素材是否被正确归组。
3. 把每日「待沉淀」玩法资产/变种回写到飞书云文档，避免规则兜底越来越吃力。
4. 让反馈训练链路先稳定产出周报，再考虑把接受概率写回主表作为辅助排序字段。
5. 继续保留看板人工抽检：抓取漏素材、封面误杀、老玩法新迭代误判，是当前三类最高风险。
